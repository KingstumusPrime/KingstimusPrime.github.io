Custom OS

A custom POSIX compatible 32 bit operating and C Library written from complete scratch. Written primarily in C but leverages assembly to complete low level tasks and boot the operating system. Contains a monolithic kernel design and supports the FAT filesystem, ATA drives, basic I/O, user mode, virtual memory, VESA video modes, and Linux-style file handling. Along with this it has a relatively mature LibC and operating systems specific toolchain capable of porting basic programs such as the famous Doom I & II.

NES Emulator

A working NES emulator written from complete scratch using C++ and SDL2 for the rendering and keyboard input. Includes a full 6502 emulator, a cycle accurate Picture Processing Unit (PPU), and a couple of debug tools such as a basic 6502 disassembler. Uses a proper object-oriented approach to emulate every single component as its own object making the codebase both easy to debug and conceptually correct.

Crown.JS

Crown.JS is a Javascript library that attempts to streamline the creation of canvas based games in Javascript. It leverages Inherence and a classical object-oriented structure to allow the user to expand upon the engine and add features needed only for that project. Features include sprites, actors, a custom input system, a smooth movement library, and dynamic camera that supports shake. In addition the project also contains a bonus multiplayer header that serves as a wrapper for PEER.JS. Allowing for multiplayer functionality in minutes.

My Journey:

Experimentation with compiler design (G--)

This project is super important to me, as it was the first project where I realized how much I loved writing low-level code. I was past wanting to know how things work; I wanted to make it work all by myself. This project ended up being is a hodgepodge of languages, including Python, C++, JavaScript, and Java. It contains a parser (JavaScript), a lexer (Python), and two VMs (C++ and Java). While I had some vision of tying it all together, I never completed the whole language toolchain.

This project was the first time I understood what a neural network actually was and how it worked. While I was mainly following tutorials, it taught me a lot about how to learn and approach things. Slime City

This project gained my a lot of fame in my Coding Club and debuted my career in multiplayer games. It taught me how to deploy a game that a ton of people would play. Custom CPU

While I got a bit of fame from the multiplayer game, I was a bit tired of writing JavaScript. Instead I wanted to expand on the skills I was starting to finally understand from G--. So after a bit of research, I came across NAND2Tetris on Coursera. I already knew a bit of Logism and did not want to learn the VHDL used in the course. So I set out on a journey to port the course to Logism blueprints. In addition to the CPU, there was also a Python assembler for the HACK assembly specification proposed in the course. All in all, this project taught me a bit about CPU design and a ton about logic gates.
